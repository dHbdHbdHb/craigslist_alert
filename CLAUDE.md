# craigslist_alert — Developer Notes

## Git workflow

The Pi and Mac both push to the same `main` branch on a schedule, so their branches frequently diverge. Follow this pattern to avoid conflicts.

### Pushing from Mac

```bash
git add <files>
git commit -m "your message"
git pull --rebase origin main && git push origin main
```

Always pull --rebase before pushing. If the Pi pushed while you were committing, the rebase replays your commit on top cleanly.

### Pulling on the Pi

The Pi always has unstaged changes (logs, CSVs, etc.) that block a plain `git pull`. Always stash first:

```bash
git stash && git pull --rebase origin main && git stash pop
```

### Deploying a change to the Pi

Make and push the change on the Mac (see above), then on the Pi:

```bash
git stash && git pull --rebase origin main && git stash pop
```

### Manually running scripts on the Pi

```bash
# Scrape new listings
bash shell_scripts/run_scraper.sh

# Send digest email (respects last_digest_date.txt — skips if already sent today)
bash shell_scripts/run_alert.sh

# Force re-send digest (e.g. after an outage)
echo "$(date -d yesterday '+%Y-%m-%d')" > ~/craigslist_alert/last_digest_date.txt && bash shell_scripts/run_alert.sh

# Commit and push data to GitHub
bash shell_scripts/upload_csv.sh
```

## Cron schedule (Pi)

| Schedule | Script | Purpose |
|---|---|---|
| every 10 min | `run_scraper.sh` | Scrape Craigslist listings |
| 2-59/10 min | `upload_csv.sh` | Commit & push data to GitHub |
| 5 7,10,16,22 | `run_alert.sh` | Send priority alerts + daily digest |

Logs go to `logs/scraper.log`, `logs/git.log`, `logs/alert.log`.

## Common failure modes

**Digest skipped with "already sent today" but no email received**
The date in `last_digest_date.txt` got written during a failed run (e.g. Pi was offline). Reset it:
```bash
echo "$(date -d yesterday '+%Y-%m-%d')" > ~/craigslist_alert/last_digest_date.txt
```

**`git pull` fails with "Need to specify how to reconcile divergent branches"**
`upload_csv.sh` uses `git pull --rebase` to handle this automatically. If you see it in the logs it means an older version of the script is running — check that the Pi has the latest `upload_csv.sh`.

**Gmail `SMTPAuthenticationError`**
The app password is expired or was invalidated (e.g. after a Google account password change). Generate a new one at myaccount.google.com/apppasswords, update `GMAIL_APP_PASSWORD` in `config.py`, and push/deploy to the Pi.
